Getting Started

Get up and running in minutes

1

Install Plumer CLI

Navigate to your Flutter project and download Plumer:

# Navigate to your Flutter project
cd your_flutter_project

# Download Plumer
wget https://raw.githubusercontent.com/denisdevke/plumer/main/plumer -O plumer

# Make it executable
chmod +x plumer

# Optional: Add alias for global access
alias plumer="dart $(pwd)/plumer"
2

Initialize Your Project

Initialize your Flutter project with GetX, Dio, and folder structure:

././plumer init

This will set up the project structure, install GetX and Dio, auto-generate recommended folder structure, and create beautiful themes!

3

Start Building

Use plumer commands to generate your app components:

# Create a complete page with controller, binding, and screen
././plumer make:page User/Profile

# Or create individual components
././plumer make:controller Auth/Login
././plumer make:screen Settings/Theme
././plumer make:service Api/User

# Install ready-made widgets
././plumer install:widget slider
././plumer install:widget button lib/components

Commands

Available Commands

Complete reference for all Plumer CLI commands and their usage.

./plumer init

Initializes your Flutter project with GetX, Dio, and beautiful theming support.

Creates: Folder structure, themes, routes, and a welcome screen

./plumer make:page [path]

Creates a complete page with controller, binding, screen, and route registration.

Example: ./plumer make:page User/Profile

./plumer make:controller [path]

Generates a GetX controller class for state management.

Example: ./plumer make:controller Auth/Login

./plumer make:binding [path]

Creates a GetX binding class for dependency injection.

Example: ./plumer make:binding Settings/Theme

./plumer make:screen [path]

Generates a Flutter screen/page widget with themed UI components.

Example: ./plumer make:screen Dashboard/Analytics

🆕 ./plumer make:model [path]

Generates an ORM-style model class extending BaseModel with CRUD operations, fillables, validation, and API integration.

Example: ./plumer make:model User
Laravel-inspired model with save(), update(), delete(), find(), fillables, validation, and automatic API calls

🆕 ./plumer make:service [path]

Creates a complete API service with Dio integration, authentication, error handling, and logging.

Example: ./plumer make:service Api/User
Creates CRUD operations, file uploads, and search functionality automatically

🆕 ./plumer install:widget [name] [folder?]

Downloads and installs ready-made widgets from the Plumer widget repository.

Examples:
./plumer install:widget slider
./plumer install:widget button lib/components

📁 Path Structure

Use forward slashes to organize your components into folders.

Example: User/Profile creates files in lib/*/user/

Service Usage

How to Use Generated Services

Complete examples showing how to integrate and use API services in your controllers and screens.

1. Generate a Service

# Generate User service
./plumer make:service Api/User

Creates:

  • • lib/services/api/user_service.dart
  • • Complete CRUD operations
  • • File upload functionality
  • • Search capabilities
  • • Error handling & logging

2. Register Service

Add to your main.dart or binding:

// In main.dart or initial binding
Get.put(UserService());

// Or lazy initialization
Get.lazyPut(() => UserService());

3. Use in Controller

class UserController extends GetxController {
  final UserService _userService = Get.find();
  var users = [].obs;
  var isLoading = false.obs;

  Future<void> loadUsers() async {
    try {
      isLoading.value = true;
      final response = await _userService.getUserList();
      users.value = (response.data['data'] as List)
          .map((user) => User.fromJson(user))
          .toList();
    } catch (e) {
      Get.snackbar('Error', 'Failed to load users');
    } finally {
      isLoading.value = false;
    }
  }
}

4. Available Methods

getUserList()

Get paginated list with filters

getUserById(id)

Get single user by ID

createUser(data)

Create new user

updateUser(id, data)

Update existing user

deleteUser(id)

Delete user by ID

Upload file for user

Search users by query

5. Complete CRUD Example

class UserController extends GetxController {
  final UserService _userService = Get.find();

  // Create user
  Future<void> createUser(Map<String, dynamic> userData) async {
    try {
      final response = await _userService.createUser(userData);
      Get.snackbar('Success', 'User created successfully');
      loadUsers(); // Refresh list
    } catch (e) {
      Get.snackbar('Error', 'Failed to create user');
    }
  }

  // Search users
  Future<void> searchUsers(String query) async {
    try {
      final response = await _userService.searchUsers(query);
      users.value = (response.data['data'] as List)
          .map((user) => User.fromJson(user))
          .toList();
    } catch (e) {
      Get.snackbar('Error', 'Search failed');
    }
  }

  // Upload user avatar
  Future<void> uploadAvatar(String userId, String filePath) async {
    try {
      await _userService.uploadUserFile(userId, filePath);
      Get.snackbar('Success', 'Avatar uploaded successfully');
    } catch (e) {
      Get.snackbar('Error', 'Upload failed');
    }
  }
}

ORM Models

Laravel-Style Models for Flutter

Complete ORM-style models with CRUD operations, fillables, validation, and automatic API integration.

1. Generate ORM Model

# Generate User model
./plumer make:model User

# Generate nested model
./plumer make:model Product/Category

Creates:

  • • lib/models/user_model.dart
  • • lib/models/base_model.dart
  • • CRUD operations (save, update, delete)
  • • Fillable fields and mass assignment
  • • Validation with error handling
  • • Automatic API integration

2. CRUD Operations

// Create and save
final user = User();
user.fill({'name': 'John', 'email': 'john@example.com'});
await user.save(); // POST /users

// Find by ID
final user = await User.find(1); // GET /users/1

// Update
user?.set('name', 'Jane');
await user?.update(); // PUT /users/1

// Delete
await user?.delete(); // DELETE /users/1

3. Fillables & Mass Assignment

class User extends BaseModel {
  String? name;
  String? email;
  bool isActive;

  @override
  List<String> get fillable => [
    'name', 'email', 'is_active'
  ];

  @override
  String get endpoint => '/users';
}

// Safe mass assignment
user.fill(formData); // Only fillable fields assigned

5. Controller Integration

class UserController extends GetxController {
  var users = <User>[].obs;
  var isLoading = false.obs;

  @override
  void onInit() {
    super.onInit();
    loadUsers();
  }

  // Load users with ORM
  Future<void> loadUsers() async {
    isLoading.value = true;
    users.value = await User.all(); // API call handled automatically
    isLoading.value = false;
  }

  // Create user with validation
  Future<void> createUser(Map<String, dynamic> data) async {
    final user = User();
    user.fill(data); // Mass assignment with fillables
    
    if (!user.isValid()) {
      Get.snackbar('Error', user.firstError ?? 'Validation failed');
      return;
    }

    if (await user.save()) { // Automatic POST /users
      users.add(user);
      Get.snackbar('Success', 'User created successfully');
    }
  }

  // Update user
  Future<void> updateUser(User user, Map<String, dynamic> data) async {
    user.fill(data);
    if (await user.update()) { // Automatic PUT /users/:id
      users.refresh(); // Update reactive list
    }
  }

  // Delete user
  Future<void> deleteUser(User user) async {
    if (await user.delete()) { // Automatic DELETE /users/:id
      users.remove(user);
    }
  }
}
